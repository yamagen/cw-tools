# CW methods restored in cw 0.9.0

`cw` now supports the historical `-M` calculation switch.

## Methods

### `-M 1`: basic CW

```text
CW = (1 + ln(ctf)) * sqrt(idf1 * idf2)
```

This is the compact explanatory formula retained from cw 0.7.0.
It can be used with or without `-k`.

### `-M 7`: waka-graph adjusted CW (default)

```text
CW = (1 + log10(ctf)) * sqrt(idf1 * idf2)
     --------------------------------------
              1 + log10(key_fq)
```

This restores the default formula from the earlier `cw.c`.
`key_fq` is the total local token frequency of all patterns matching the
`-k` regular expression inside the selected units.  For an exact key pattern,
it is the local frequency of that pattern.

The denominator is common to every pair in one key-selected graph.  It adjusts
the scale between graphs made from keys of different frequencies.  The use of
`log10` also makes the pair-frequency effect gentler than the basic formula's
natural logarithm.

### `-M 12`: rare-pattern experimental CW

```text
CW = (1 + key_fq / ctf) * sqrt(idf1 * idf2)
```

This restores historical method 12.  To reproduce the old C implementation,
`key_fq / ctf` is intentionally integer division before conversion to a
floating-point value.  It strongly rewards pairs that occur rarely inside the
key-selected set.

### `-M 16`: global-pair adjusted CW

```text
CW = (1 + ln(N / global_pair_df))
     * sqrt(idf1 * idf2)
     * (1 + ln(local_ctf))
```

This restores the structure of historical method 16, which was marked
`best` in the earlier source.  In the current pipeline, `global_pair_df` is
the number of all input units containing the pair, while `local_ctf` is the
number of key-selected units containing it.  `N / global_pair_df` is computed
with floating-point division; the accidental integer truncation of the old C
expression is not reproduced.

This current interpretation maps the old `gq` idea onto the broad input
corpus.  The historical program could also use a broader key-containing set
as `gq` when a narrower anthology range was selected, so exact archival
reproduction of that nested scope would require a separate scope option.

## Requirements

Methods 7 and 12 require `-k`, because their formulas contain `key_fq`.
They also require pair input containing token frequencies, generated by
`pair 0.2.0` or later.  Methods 1 and 16 accept the older three-column pair
format.  Method 16 may be used without `-k`; in that case local and global
scopes are both the complete input corpus.

## Examples

```sh
grep '^1' tests/data/hachidaishu-pair.txt |
  ./pair |
  ./cw -p 2,3 -k '^梅/名$' -M 7
```

```sh
grep '^1' tests/data/hachidaishu-pair.txt |
  ./pair |
  ./cw -p 2,3 -k '^梅/名$' -M 12
```

`-M 7` is the default, so the first command may omit `-M 7`.

```sh
grep '^1' tests/data/hachidaishu-pair.txt |
  ./pair |
  ./cw -p 2,3 -k '^梅/名$' -M 16
```
